# Text Lock: Phone Call Record Management

Audience: business users considering whether kintone can manage phone call records and follow-up status.

Language for this prototype: English.

## 00_pain

- **Claim**: Phone calls disappear when records stay on paper.
- **Support**: Details are written quickly, follow-up is handled by memory, and managers cannot see the trend.
- **Speaker note**: Start from a familiar scene: the call is handled, but the record is not reusable.
- **Must keep**: phone calls, paper records, follow-up.
- **Route guidance**: Route C should use staged fragments to show scattered call information.

## 01_system

- **Claim**: kintone turns every call into a shared record.
- **Support**: Caller, topic, status, owner, and next action sit in one visible table.
- **Speaker note**: Shift from emotional pain to a clear operational object.
- **Must keep**: kintone, shared record, status, next action.
- **Route guidance**: Route C should show an interface-like record board without relying on screenshot fidelity.

## 02_outcome

- **Claim**: Follow-up becomes easier when the whole team sees the same history.
- **Support**: Open items, resolved calls, and repeated issues become visible across the team.
- **Speaker note**: Close with the operational benefit: fewer missed handoffs and better reuse.
- **Must keep**: follow-up, same history, team.
- **Route guidance**: Route C should use reveal/animation to show flow from call to record to completed follow-up.

# Storyboard: AB Route Test

## 00_cover

- **Title**: From field feedback to improvement loop
- **Core claim**: Turn scattered field feedback into one visible improvement loop.
- **Primary job**: Establish the shared theme for both Route A and Route B.
- **Support points**: Field reports, approval, notifications, and summaries are connected by one work loop.
- **Visual metaphor**: A field worker, a manager, and a yellow helper around a calm loop.
- **Speaker notes**: This opens the story with a concrete operational loop rather than a feature list.
- **B asset mode**: continuity-locked for recurring worker/manager/helper; identity-locked for any kintone mark.

## 01_pain

- **Title**: Separate updates hide the pattern
- **Core claim**: When updates live in separate places, the team loses the pattern.
- **Primary job**: Show why the current state is hard to improve from.
- **Support points**: Paper, spreadsheets, and chat messages each capture fragments but not the trend.
- **Visual metaphor**: The same worker looks stressed while updates scatter across paper, chat, and spreadsheet fragments.
- **Speaker notes**: The point is not that people are careless; the workflow itself makes learning difficult.
- **B asset mode**: continuity-locked for the worker; free-derivative for generic documents/messages.

## 02_solution

- **Title**: One app connects the loop
- **Core claim**: One app can connect input, approval, notice, and summary.
- **Primary job**: Make the operating loop legible.
- **Support points**: Input -> Approval -> Notice -> Summary.
- **Visual metaphor**: Four clear stations connected by an improvement loop with a friendly helper.
- **Speaker notes**: This slide should make the operating loop visible at a glance.
- **B asset mode**: continuity-locked for helper; free-derivative for generic process icons; identity-locked for kintone mark.

## 03_continue

- **Title**: Every small fix leaves a trace
- **Core claim**: Improvement continues when every small fix leaves a trace.
- **Primary job**: Close with continuity and next action.
- **Support points**: History, review, reuse, and expansion across teams.
- **Visual metaphor**: The worker and manager now look relieved, reviewing a simple trail of small improvements.
- **Speaker notes**: Close by shifting from single issue resolution to repeatable operational learning.
- **B asset mode**: continuity-locked for recurring people/helper; free-derivative for trace/checkpoint motifs.

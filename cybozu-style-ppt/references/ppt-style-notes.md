# Cybozu/Kintone PPT Style Notes

These notes summarize the initial HTML-skill material for PPT use.

## Source Insight

The teammate's HTML workflow emphasized that the slow parts are not copywriting, but structure tuning, matching official-site taste, repeated visual adjustment, and producing a demo that others can review. The same applies to PPT: the skill should reduce repeated decisions about information architecture and visual language.

## Inputs To Request

Required:

- Product or service name, including Chinese, Japanese, and English names if available.
- Core message in 1-3 sentences, especially the strongest difference from alternatives.
- Target customer: industry, decision maker, company size, region, and business context.
- Desired action after the deck: request materials, book demo, start trial, approve proposal, align internally, or another next step.
- Deck language: Japanese, Chinese, bilingual, English, or mixed.

Helpful:

- Existing page or section ideas.
- Case studies, pricing, awards, certifications, security, compliance, or implementation facts.
- Brand guidelines, VI, logo, color, fonts, product screenshots, or approved illustrations.
- Competitors or comparable services.
- Existing URLs and specific pages to reference.

## PPT Translation

Convert long-page web patterns into slide-native structures:

- Hero area -> title slide or product promise slide.
- Category navigation -> agenda, audience segmentation, or solution map.
- Diagnosis section -> assessment, maturity model, or customer type slide.
- Pain cards -> problem overview slide with concise labels.
- Solution flow -> workflow slide with editable arrows, containers, and icon groups.
- Demo/product UI -> editable dashboard mockup unless real screenshot fidelity is required.
- CTA -> closing slide with next steps.

## Style Observations From Screenshots

- Warm yellow is the main brand accent for tabs, buttons, section markers, and borders.
- Black, bold headings carry the message; body copy is short and practical.
- White and pale cream backgrounds create a soft business feel.
- Cards use light borders, mild rounding, and generous internal spacing.
- The visual tone mixes business utility with friendly illustrations and simple iconography.
- Section labels can use yellow ribbon/notch motifs, but should be simplified for slides.
- Layouts should feel modular and editable, especially dashboard panels, card grids, and workflows.

## PPT-Specific Guardrails

- Build text, panels, mockups, charts, and diagrams as native objects.
- Do not import an HTML page screenshot as the slide design.
- Keep one dominant message per slide.
- Prefer fewer, larger modules over webpage density.
- Check multilingual line breaks manually, especially Japanese and Chinese headings.
- Keep footer or navigation motifs subtle; PPT does not need persistent web navigation.
